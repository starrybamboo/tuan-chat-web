# 空间侧边栏知识库

## 当前结构（2026-03）

- 通用 `sidebarTree` 仍只承载：
  - 频道
  - 文档
- 聊天室空间侧边栏中的“素材包”不是 `sidebarTree` 新节点类型，而是 `ChatRoomListPanel` 中追加渲染的独立区域。

## 素材包区域

- 区域标题：`素材包`
- 位置：现有频道/文档分类之后
- 入口行为：
  - 右侧 `+` 打开“导入局外素材包”弹窗
  - 导入后刷新当前 space 的局内素材包列表
- 展示层级：
  - `素材包区域`
  - `局内素材包名称`
  - `素材项`

## 拖拽发送

- 真正可拖拽的是“素材项”，不是整个局内素材包。
- 素材项拖拽使用独立 payload 协议：
  - `app/components/chat/utils/materialItemRef.ts`
- 房间接收层复用现有 `RoomDocRefDropLayer`，新增素材项识别与 drop 发送。
- drop 到聊天室后：
  - 读取素材项中的消息模板数组
  - 映射为一个或多个 `ChatMessageRequest`
  - 按顺序调用现有发送链路发送到当前房间

## 设计取舍

- 之所以不直接扩展 `sidebarTree` 为三层或多类型节点，是为了控制本轮改动范围：
  - 避免波及现有频道/文档排序、右键菜单、增删改、归一化逻辑
  - 先把“导入局外素材包 -> 展开查看素材项 -> 拖拽发送”这条主链路跑通
- 后续如果素材包成为空间侧边栏核心对象，再考虑将其并入统一树模型。
