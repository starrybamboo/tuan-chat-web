# 移动端消息能力与 Web 端核对

> 核对日期：2026-07-19。
>
> 范围：房间消息主链路，以及紧邻消息区的线索、地图、战斗、导入导出和 WebGAL 工作流。不包含账号、空间/房间管理、角色管理、通知和私聊的整站对齐。
>
> 判定规则：必须同时存在用户入口、状态处理、请求/缓存链路和可见结果，才算「已实现」。只有类型、底层函数或未接 UI 的 handler，不算用户可用能力。

## 结论

旧版「已实现 26 / 降级 12 / 没有 3」的统计口径过粗，不能反映真实缺口。按原子能力重新拆分后，当前消息工作台至少有：

| 结论 | 数量 | 说明 |
| --- | ---: | --- |
| 降级 | 13 | 移动端有入口或展示，但范围、交互或语义弱于 Web。 |
| 没有 | 39 | 移动端没有用户可达入口，或完全没有对应状态/处理链路。 |
| 未完全对齐 | 52 | 仅统计本文范围内已经从 Web 源码确认的能力。 |

移动端已经稳定具备文本发送、图片发送、表情、指令请求、规则执行、Dice/状态事件、聊天状态、角色与头像身份、常规消息和媒体渲染、复制、回复、文本内容编辑、删除、线索收藏、搜索、戳一戳、失败重试、消息拖拽排序、WebSocket 实时同步与本地缓存。这些核心链路不在下面重复展开。

## 已确认的关键代码事实

- `ChatComposer.tsx` 的附件按钮固定调用 `MOBILE_MESSAGE_ATTACHMENT_KIND.IMAGE`；视频、音频和文件虽然有 picker 与上传实现，但用户从聊天输入器无法触发。
- `MessageActionMenu.tsx` 声明了 `sendToRoom`，主聊天调用没有传 `canSendToRoom`，`ChatShell.tsx` 的 `handleMessageAction` 也没有对应分支。
- 移动端对 `annotations` 的唯一业务使用主要停留在发送 mutation 的可选参数；消息列表没有标注选择器、标注条或特效预览。
- 移动端已有 `position` 和 `patchRoomMessages`，但当前只用于移动既有消息，不能等同 Web 的插入模式或 undo/redo 历史栈。
- `MobileMessageCards.tsx` 已覆盖 Forward、State Event、WebGAL Choose、DocCard、ClueCard、RoomJump 的展示，但只有 Forward/Clue 提供局部详情，其他卡片没有 Web 同等导航或编辑 handler。

## 建议优先级

1. **先修用户入口与实际能力不一致**：补视频/音频/文件附件入口，并移除或接通悬空的 `sendToRoom`。
2. **补消息生产闭环**：合并/逐条/多房间转发、插入消息、撤销/重做、全选和范围选择。
3. **补卡片导航闭环**：DocCard 打开文档、RoomJump 跳转房间、WebGAL Choose 编辑。
4. **最后拆分演出工作流**：annotations、背景/BGM、立绘、导演特效、WebGAL 预览和消息 diff 应作为一组产品能力设计。

- `apps/mobile/src/app/role-edit.tsx` 的创建分支调用 `createRole` 时只提交 `roleName`、`description`、`type`；`selectedRuleId` 只在已有 `roleId` 的编辑分支后续使用。
- `apps/mobile/src/features/roles/edit/AvatarGrid.tsx` 的 `handlePickImage` 从多选结果取 `attachments[0]`，再进入单图裁剪和上传；没有 Web 的上传队列、源图替换、变换和立绘裁剪状态。
- `apps/mobile/src/features/roles/edit/RuleSection.tsx` 只调用规则分页和详情查询；移动端 `features/roles` 没有规则创建、更新、删除或克隆 mutation。
- `apps/web/app/components/Role/CharacterDetail.tsx` 明确接入音频、骰娘关联、骰娘 JSON、规则选择和 ST 导入；移动端角色编辑页均没有对应入口。
- `apps/mobile/src/features/roles/roleAvatarVariantGroups.ts` 只提供按服务器返回数据分组的纯函数，不能证明移动端具备立绘组 CRUD 或分配能力。

### 角色模块优先级

1. **先补角色编辑基本闭环**：创建时初始化规则/能力、角色页选择房间、头像批量上传与源图替换。
2. **再补角色资产管理**：立绘组 CRUD、分配/批量套用、头像标题、立绘/头像高级裁剪和变换。
3. **最后补规则与骰娘工作台**：规则模板编辑/克隆/删除、ST 导入、骰娘文案/JSON、语音和骰娘关联、角色预览。
