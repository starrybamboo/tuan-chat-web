# 移动端消息能力对 Web 端简版核对表

> 口径：以 Web 端消息能力为基准，只标「已实现 / 降级 / 没有」。
>
> 阅读方式：
> - **已实现**：移动端已有同等核心能力，不再展开差异。
> - **降级**：只写 Web 端有、移动端缺或弱化的点。
> - **没有**：移动端当前没有对应入口或处理链路。

## 降级

| 功能 | Web 端有 | 移动端现状 |
| --- | --- | --- |
| 骰子/状态指令 | Web 端完整接入房间发送链路、状态事件、dice feedback 和规则能力更新。 | 移动端能执行核心 dicer 和状态事件，但规则覆盖、隐藏骰、复杂反馈链路需要专项核对。 |
| @ 提及 | Web 端是富文本 mention，维护 `mentionedRoles` 和去 mention 文本。 | 移动端是纯文本 `@` 角色列表插入，未发现同等结构化 mention 发送链路。 |
| Dice 消息 | Web 端有 Dice / diceTurn 专门渲染和部分编辑能力。 | 移动端能显示和执行结果，但未对齐 Web 的完整 Dice 编辑/隐藏骰交互。 |
| Forward 卡 | Web 端能创建合并/逐条转发消息。 | 移动端只能渲染 Forward 卡，未发现创建转发入口。 |
| WebGAL Choose | Web 端可渲染并编辑 WebGAL 选项消息。 | 移动端只读展示选项摘要，未发现编辑或演出控制入口。 |
| 文档卡/线索卡/房间跳转 | Web 端与文档、线索、房间侧栏深度联动。 | 移动端能显示卡片，但偏只读/轻交互。 |
| 编辑文本 | Web 端多类消息可内联编辑。 | 移动端主要是把消息内容放回输入框再提交，更接近纯文本编辑。 |
| 多选 | Web 端多选后可做删除、转发等批量操作。 | 移动端支持多选、批量复制、批量删除，但未发现多选转发/批量收藏。 |
| 发送到当前房间 | Web 端有完整转发窗口，可选择目标房间。 | 移动端线索面板能发送线索到当前房间，但消息长按菜单的 `sendToRoom` 没接主聊天处理分支。 |
| 消息标注 annotations | Web 端可选择标注，并随消息发送。 | 移动端底层发送接口可带 annotations，但没有移动端标注选择/发送入口。 |
| 背景/BGM/清理语义 | Web 端可设置/取消背景、BGM、清理场景语义。 | 移动端能显示已有 BGM 语义，但不能像 Web 一样设置或取消。 |
| WebGAL 实时演出 | Web 端有实时渲染、跳转、重渲染、预览控制。 | 移动端只读 WebGAL Choose 卡，没有实时演出控制。 |
| 文档编辑器/素材库/导入导出 | Web 端有文档编辑、素材库、聊天导入、Premiere 导出等工作台能力。 | 移动端消息区只有卡片和轻量面板，不是完整创作工作台。 |
| 线索面板 | Web 端线索文件夹和公共线索能力更完整。 | 移动端是右抽屉轻量线索面板。 |
| 地图/战斗面板 | Web 端房间扩展面板更完整。 | 移动端有地图/战斗入口，但深层能力未与 Web 完全对齐。 |

## 没有

| 功能 | Web 端有 | 移动端缺口 |
| --- | --- | --- |
| 插入消息 | Web 端可进入插入模式，把新消息插到指定消息后。 | 移动端没有插入模式、`insertAfterMessageId` 状态或对应发送分支。 |
| 消息撤销/重做 | Web 端有消息 undo/redo 栈，并通过 patch 恢复历史操作。 | 移动端没有撤销/重做入口或对应状态。 |
| 合并/逐条/多房间转发 | Web 端支持合并转发、逐条转发、选择多个目标房间。 | 移动端没有创建转发消息或选择目标房间的入口。 |

## 优先补齐建议

1. **先补消息转发**：包括发送到当前房间、合并/逐条、多房间目标选择。
2. **再补插入消息与撤销/重做**：这是 Web 消息编辑体验的核心差异。
3. **最后补 annotations / 背景 / BGM**：这块牵涉 WebGAL 演出语义，适合和演出能力一起设计。
