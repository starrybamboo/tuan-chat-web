# 变更提案: AI 生图 Web 默认切回同源代理

## 需求背景

Web 端“直连” NovelAI 会被浏览器跨域/CORS（含预检 OPTIONS）拦截，导致模型拉取与生图请求失败。为保证可用性，需要将 Web 默认请求方式切回“同源代理”。

## 变更内容

1. Web 环境默认请求方式：`同源代理`
2. 增加本地持久化：刷新页面后仍保持上次选择
3. 保留“直连”作为可选开关（用于排查上游响应，但可能被浏览器拦截）

## 影响范围

- **模块:** `app`
- **文件:**
  - `app/routes/aiImage.tsx`
  - `helloagents/wiki/modules/app.md`
  - `helloagents/CHANGELOG.md`
- **API:** 无
- **数据:** 无

## 核心场景

### 需求: Web 默认可用
**模块:** `app`

#### 场景: 刷新页面后仍可用
在 `/ai-image` 页面刷新后：
- 默认使用“同源代理”模式请求
- 不会因默认直连触发 CORS 导致无法请求

## 风险评估

- **风险:** 无（仅调整默认与持久化行为）
- **缓解:** 保留“直连”手动开关作为备选
