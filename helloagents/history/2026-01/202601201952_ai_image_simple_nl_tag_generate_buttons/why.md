# 变更提案: AI 生图（普通模式）一键出图 + 按 tag 出图

## 需求背景
当前普通模式的生图流程本质为：自然语言 → tags（prompt）→ 生图。用户在“想做一张新图”时主要使用自然语言描述，但现有交互更偏向“先得到 tags 再点生成”，导致频繁新图生成时操作成本高。

同时，用户在编辑 tags 后希望可以明确地“按 tags 直接出图”，而不是再次触发自然语言转换。

## 变更内容
1. 普通模式将“生成”交互拆分为两个明确入口：
   - 自然语言输入框右侧提供“一键出图”：执行 NL→tags→出图，并将最终 tags 写回 tags 区（便于继续微调）
   - tags 区提供“按 tag 出图”：不触发 NL→tags，直接用当前 tags 出图
2. 删除普通模式主视图区的底部“生成”按钮，减少冗余并节省空间（专业模式不受影响）。

## 影响范围
- **模块:** `ai-image`
- **文件:**
  - `app/routes/aiImage.tsx`
- **API:** 无
- **数据:** 无

## 核心场景

### 需求: 普通模式快速自然语言出图
**模块:** ai-image
用户以自然语言为主进行新图生成，需要最低摩擦的一键出图入口。

#### 场景: 输入自然语言后一键出图
在普通模式中：
- 预期结果：自然语言输入框右侧提供“一键出图”按钮
- 预期结果：点击“一键出图”会执行 NL→tags→出图
- 预期结果：转换得到的 tags（合并画风 tags 后）会写回 tags 区

### 需求: 普通模式按 tags 直接出图
**模块:** ai-image
用户修改 tags 后，应可直接按 tags 出图，不应触发自然语言转换。

#### 场景: 编辑 tags 后按 tag 出图
在普通模式中：
- 预期结果：tags 区显示“按 tag 出图”按钮
- 预期结果：点击后直接用当前 tags 出图（并合并已选画风 tags）

## 风险评估
- **风险:** 入口拆分后按钮逻辑易出现“写回未同步/生成未合并画风”问题
- **缓解:** 统一在生成入口处合并画风 tags，并在生成前写回 `prompt/negativePrompt`
