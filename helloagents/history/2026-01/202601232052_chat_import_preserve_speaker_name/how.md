# 技术设计: 聊天导入保留原发言人名

## 技术方案
### 核心技术
- React 19 / TypeScript
- 房间消息发送流程（RoomWindow）

### 实现要点
- 扩展导入消息传参，将 speakerName 传入导入发送逻辑。
- 在导入发送时，优先使用导入的 speakerName 写入 `request.webgal.customRoleName`，仅在非空时设置。
- 保留现有骰娘解析、旁白与立绘位置逻辑；customRoleName 不影响角色绑定与消息类型。

## 安全与性能
- **安全:** 对 speakerName 做 trim，避免空白覆盖。
- **性能:** 每条消息多一次字符串处理，性能影响可忽略。

## 测试与部署
- **测试:** 手工导入包含多角色/骰娘/旁白的文本，确认显示名与角色绑定一致。
- **部署:** 前端构建发布，无额外步骤。
