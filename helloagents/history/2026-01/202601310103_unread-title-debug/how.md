# 技术设计: 未读标题调试日志

## 技术方案
### 核心技术
- React Hooks
- 控制台日志（受 localStorage 开关控制）

### 实现要点
- 在 `chatPage` 中新增调试开关读取（如 `localStorage["tc:unread:debug"]`）。
- 当总未读/私聊未读/好友申请数变化时输出：
  - 标签未读汇总值
  - 私聊各联系人未读数与聚合来源
  - 群聊未读汇总值（`unreadMessagesNumber`）
  - pending 好友请求数量
- 仅在开关开启时输出，默认关闭。

## 架构设计
无。

## API 设计
无。

## 数据模型
无。

## 安全与性能
- **安全:** 无新增风险。
- **性能:** 日志仅在调试开关开启时触发。

## 测试与部署
- **手动:** 开启调试开关后刷新 `/chat/private`，检查控制台输出是否包含未读来源分解。
- **类型检查:** 运行 `pnpm typecheck`。
